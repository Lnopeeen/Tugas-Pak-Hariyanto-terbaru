Fully Developed Use Case Description

Use Case: Owner Login ke Dalam Sistem

Aktor: Owner

Deskripsi: Owner memasukkan username dan password untuk mengakses sistem pengelolaan toko daging.

Alur Normal:

  1. Owner membuka halaman login.
  2. Owner mengisi username dan password.
  3. Sistem memverifikasi kredensial.
  4. Jika berhasil, sistem menampilkan dashboard.

Pre-condition: Owner memiliki akun aktif.

Post-condition: Owner masuk ke sistem dan dapat mengakses fitur lainnya.

Use Case: Transaksi

Aktor: Owner

Deskripsi: Owner melihat daftar transaksi pembelian daging dari pelanggan.

Alur Normal:

  1. Owner klik menu "Transaksi".
  2. Sistem menampilkan daftar transaksi.
  3. Owner dapat melihat status transaksi: Belum Konfirmasi, Konfirmasi, Terkirim, Diterima.

Alur Alternatif (<>):

  • Belum Konfirmasi: Transaksi belum diverifikasi pembayaran.
  • Konfirmasi: Pembayaran telah dikonfirmasi.
  • Terkirim: Produk dalam proses pengiriman.
  • Diterima: Produk telah diterima pelanggan.

Pre-condition: Owner sudah login ke sistem.

Post-condition: Data transaksi berhasil ditampilkan.

Use Case: Kelola Produk

Aktor: Owner

Deskripsi: Owner dapat menambah, mengedit, dan menghapus produk daging.

Alur Normal:

  1. Owner klik menu Produk.
  2. Sistem menampilkan daftar produk.
  3. Owner dapat memilih untuk menambah produk baru atau mengedit produk yang ada.
  4. Owner menyimpan perubahan.

Relasi:

  • `<>` Add New Product
  • `<>` Edit, Delete
  • `<>` Save
Use Case: Galeri & Kategori

Aktor: Owner

Deskripsi: Owner mengatur galeri gambar dan kategori produk.

Fitur Galeri:

  • Melihat daftar galeri
  • Menambah galeri baru (`<>` Add New Galleries)
  • Mengedit dan menghapus galeri (`<>` Edit, Delete)
  • Simpan (`<>` Save)

Fitur Kategori:

  • Melihat daftar kategori
  • Menambah kategori baru (`<>` Add New Categories)
  • Mengedit dan menghapus kategori (`<>` Edit, Delete)
  • Simpan (`<>` Save)
