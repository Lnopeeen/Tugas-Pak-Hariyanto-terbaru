Fully Developed Use Case Description - Toko Daging

  • Aktor: Owner
  • Deskripsi: Owner melakukan login ke dalam sistem menggunakan kredensial yang valid.
  • Prekondisi: Owner telah memiliki akun.
  • Alur Normal:
    1. Owner membuka halaman login.
    2. Owner mengisi username dan password.
    3. Sistem memverifikasi kredensial.
    4. Owner masuk ke dalam sistem.
  • Postkondisi: Owner berhasil login dan dapat mengakses fitur sistem.

  • Aktor: Owner
  • Deskripsi: Owner mengakses dan mengelola transaksi pelanggan dari sidebar.
  • Prekondisi: Owner telah login ke sistem.
  • Alur Normal:
    1. Owner memilih menu "Transactions" di sidebar.
    2. Sistem menampilkan daftar semua transaksi.
    3. Owner dapat melihat detail transaksi berdasarkan status:
      • Belum Konfirmasi
      • Konfirmasi
      • Terkirim
      • Diterima
  • Postkondisi: Owner mendapatkan informasi status transaksi dan dapat menindaklanjuti jika diperlukan.

  • Aktor: Admin
  • Deskripsi: Admin menambahkan, mengedit, atau menghapus produk daging di sistem.
  • Alur Normal:
    1. Admin memilih menu "Product".
    2. Admin dapat:
      • Menambahkan produk baru
      • Mengedit produk
      • Menghapus produk
    3. Sistem menyimpan perubahan.
  • Postkondisi: Data produk diperbarui.
